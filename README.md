# ml_project_building_risk


## Тема проекта:
"Прогнозирование рисков превышения бюджета строительных проектов на основе открытых данных".

# Актуальность.
По статистике большинство строительных проектов превышают изначально заложенный в них бюджет. ML-решения позволяют выявлять риски на начальных этапах планирования, что ведёт к существенной экономии в процессе строительства объектов.

# Новизна.
Существуют аналоги, которые по своей сути являются дорогим решением и требуют опреденной квалификации в донастройке спецификаций под определенные проекты. Текущее решение предлагает легковесный ML-инструмент "из коробки", с возможностью простого масштабирования и объяснением причин риска посредством SHAP-анализа.

1.1 Организационная структура и цели 
Понимание бизнес-контекста

1.2 Оценка текущей ситуации и риски 
Анализ ресурсов и зон неопределенности

1.3 Data Mining цели 
Постановка ML-задачи и метрик

1.4 План проекта 
Навыки планирования и декомпозиции

📍 Раздел 1.1. Организационная структура и цели

Заказчик:
Региональный застройщик. Данные позволяют обосновать этот выбор, так как дают представление о масштабах реальных компаний.

Пользователи:
Директор по строительству, руководитель проектного офиса, финансовый контролер

Бизнес-цель:
Снижение превышения бюджета строительных проектов за счет раннего выявления высокорисковых объектов на этапе планирования

Существующие решения:
Экспертная оценка и исторические данные — работают плохо, так как не учитывают комплекс факторов, а методы оценки часто являются линейными и не учитывают множество других факторов в процессе решения проблем. Открытые данные из сети интернет позволяют создать объективную, масштабируемую систему.

📍 Раздел 1.2. Текущая ситуация и риски

Железо:
Облачные ресурсы Yandex Cloud / Google Colab. Закупка железа не требуется.

Данные:
Открытые данные из сети интернет (достаточно объёмные, чтобы построить MVP: характеристики проектов, портфели застройщиков, региональная статистика). Доступ через скачивание CSV-файлов.

📍 Раздел 1.3. Решаемые задачи и метрики (Data Mining goals)

Тип задачи:
Бинарная классификация: прогноз риска превышения бюджета (1 — превысит, 0 — уложится)

Целевая переменная (что предсказываем): 
Создадим ее сами

Метрика качества:
AUC-ROC(дополнительно F1-Score).

Критерий успеха:
Модель может использоваться как ранняя система предупреждения в портфеле проектов компании.

# Стек технологий

- DVC (версионирование данных)
- MLflow (трекинг экспериментов)
- FastAPI (развертывание)
- GitHub Actions (CI/CD)

## Структура
- data/ — данные (версионируются через DVC)
- notebooks/ — исследовательский анализ
- src/ — код для обработки, обучения, API
- models/ — сохранённые модели
