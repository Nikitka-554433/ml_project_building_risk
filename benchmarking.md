## Основная функция:	
1. Предиктивная аналитика, ИИ-ассистент, прогнозирование рисков, управление KPI;
2. Сбор и обработка данных с оборудования (IIoT), MES-системы;
3. Ручная экспертиза, интуиция, «опыт прошлых лет».

## Технология:
1. ИИ, DSLM (предметно-ориентированные языковые модели), LLM;
2. Платформа ZIIoT, MES, промышленные датчики;
3. Человеческий фактор.

## Ключевая особенность:
1. Мобильный ИИ-помощник, управление по голосу, прогноз «что будет дальше»;
2. Сквозной сбор данных от оборудования до управленческих решений;	
3. Низкий порог входа, не требует внедрения.

## КЛЮЧЕВЫЕ ВЫВОДЫ ДЛЯ ПРОЕКТА
Рынок активно внедряет ИИ - ~30% девелоперов используют ИИ, целевой ориентир >50% .

Главный барьер — качество данных - именно это и решает настоящий проект. В нем используются открытые, структурированные данные ДОМ.РФ.

Официальные стандарты существуют - Минстрой и ДОМ.РФ разработали методику оценки по 12 показателям. Это прямые критерии для бенчмаркинга .

Технология информационного моделирования (ТИМ) растёт - 41% застройщиков уже используют. Это подтверждает цифровую зрелость отрасли .

## Конкуренты есть, но ниша остаётся — Linkage от ВестЛинк — серьёзный игрок. Но они ориентированы на крупный бизнес и сложную инфраструктуру. Текущее решение ориентировано на лёгкость, открытые данные и фокус на прогнозирование рисков.

# Вопрос 1. Какие функции являются обязательными?
На основе анализа рынка, обязательными функциями для MVP являются:
- Прогнозирование рисков превышения бюджета (бинарный классификатор);
- API для интеграции с системами заказчика;
- Базовый дашборд с отчётом по ключевым факторам риска;
- Мониторинг качества прогнозов.

# Вопрос 2. Какие функции можно предложить, чтобы отличаться?
Уникальные преимущества проекта:
- Использование только открытых данных (ДОМ.РФ, Росстат) — не требует доступа к внутренним системам заказчика;
- Понятные прогнозы (SHAP-анализ) — заказчик видит не только «красный флаг», но и причины риска;
- Фокус на средний и малый бизнес (ниша, которую игнорируют крупные ERP-системы).

# Вопрос 3. Какие недостатки конкурентов можно устранить?
- Дороговизна внедрения (Linkage и аналоги ориентированы на крупный бизнес) - наше решение - лёгкий SaaS с низким порогом входа;
- Закрытость алгоритмов («чёрный ящик») - используем SHAP для интерпретации прогнозов;
- Зависимость от внутренних данных заказчика - работаем только с открытыми данными;

# Вопрос 4. Какая стратегия монетизации наиболее эффективна?
Наиболее эффективной для целевого сегмента (средний и малый девелопер) является SaaS-модель с ежемесячной подпиской. 
Это позволяет снизить порог входа и получить быструю обратную связь от рынка. Дополнительно — Freemium (5 бесплатных прогнозов в месяц) для привлечения первых клиентов.

# Вопрос 5. Какой пользовательский опыт можно улучшить?
- Простота интеграции (REST API, понятная документация, примеры кода);
- Понятная визуализация рисков (не просто «красный/зелёный», а список факторов с их вкладом);
- Быстрый онбординг (загрузка данных о проекте за 5 минут вместо месяцев/недель внедрения)
